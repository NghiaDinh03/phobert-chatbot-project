# API Reference — CyberAI Assessment Platform v2.0

Base URL (internal): `http://backend:8000`
Base URL (via Next.js proxy): `http://localhost:3000/api`

All endpoints are accessible from the frontend via the `/api/*` rewrite proxy. Rate limiting is enforced on the backend via `slowapi`.

---

## 1. Chat API — `backend/api/routes/chat.py`

### `POST /api/chat`
Send a user message and receive an AI response with RAG context.

**Rate limit:** 10/minute

**Request body:**
```json
{
  "message": "Yêu cầu kiểm soát truy cập theo ISO 27001 là gì?",
  "session_id": "user-abc-123"
}
```

**Response:**
```json
{
  "response": "Theo Phụ lục A.9 của ISO 27001:2022...",
  "session_id": "user-abc-123",
  "route": "iso",
  "sources": ["iso27001_annex_a.md", "tcvn_11930_2017.md"],
  "used_rag": true
}
```

**Notes:**
- `session_id` is generated by the frontend (UUID) and stored in `localStorage`
- The backend loads the last 20 messages from `data/sessions/<session_id>.json` and passes them as conversation history to the LLM
- `route` reflects the category detected by `model_router.py`: `iso`, `security`, `legal`, `technical`, `news`, `general`, `greeting`
- `sources` lists the document files used as RAG context

---

### `GET /api/chat/history/{session_id}`
Retrieve the full conversation history for a session.

**Response:**
```json
{
  "session_id": "user-abc-123",
  "messages": [
    { "role": "user", "content": "...", "timestamp": "2026-03-24T08:00:00Z" },
    { "role": "assistant", "content": "...", "timestamp": "2026-03-24T08:00:05Z" }
  ],
  "message_count": 12,
  "last_accessed": "2026-03-24T08:00:05Z"
}
```

---

### `DELETE /api/chat/history/{session_id}`
Clear a chat session (removes the session JSON file).

**Response:**
```json
{ "status": "cleared", "session_id": "user-abc-123" }
```

## 2. ISO 27001 Assessment API — `backend/api/routes/iso27001.py`

### `POST /api/iso27001/assess`
Submit an assessment form and receive an AI-generated gap analysis report.

**Rate limit:** 3/minute

**Request body:**
```json
{
  "standard_id": "iso27001",
  "organization": {
    "name": "Công ty ABC",
    "industry": "finance",
    "size": "medium"
  },
  "infrastructure": {
    "has_firewall": true,
    "has_ids": false,
    "has_vpn": true,
    "cloud_provider": "aws"
  },
  "controls": {
    "A.5.1": "yes",
    "A.6.1": "partial",
    "A.8.1": "no"
  }
}
```

**Response:**
```json
{
  "id": "assess-20260324-xyz",
  "standard": "ISO 27001:2022",
  "risk_level": "high",
  "controls_analyzed": 93,
  "controls_met": 47,
  "compliance_percent": 50.5,
  "report_markdown": "## Báo cáo Đánh giá ISO 27001\n\n### Tổng quan...",
  "gaps": ["A.8.1 — Quản lý tài sản chưa có", "..."],
  "action_plan": ["1. Xây dựng chính sách...", "..."],
  "created_at": "2026-03-24T08:00:00Z"
}
```

---

### `GET /api/iso27001/history`
Return all past assessment records.

**Query params:** `limit` (default: 50)

**Response:**
```json
{
  "assessments": [
    {
      "id": "assess-20260324-xyz",
      "standard": "ISO 27001:2022",
      "organization": "Công ty ABC",
      "risk_level": "high",
      "compliance_percent": 50.5,
      "created_at": "2026-03-24T08:00:00Z"
    }
  ],
  "total": 12
}
```

---

### `GET /api/iso27001/history/{assessment_id}`
Get the full report for a specific assessment (used by the Analytics detail modal).

---

### `DELETE /api/iso27001/history/{assessment_id}`
Delete a single assessment record.

---

### `GET /api/iso27001/chromadb/stats`
Return ChromaDB vector store statistics.

**Response:**
```json

  "total_documents": 7,
  "total_chunks": 342,
  "metric": "cosine",
  "collection": "iso_knowledge",
  "store_path": "/data/vector_store"
```

**Response:**

```json
  "status": "ok",

  "documents_processed": 7
```

```json

  "results": [
    {

      "source": "iso27001_annex_a.md",

    }

```

## 3. News API — `backend/api/routes/news.py`

### `GET /api/news`
Fetch latest articles for a category (triggers RSS fetch + background translation).

**Query params:** `category` (`cybersecurity` | `stocks_vietnam` | `stocks_international`), `limit` (default: 15)

**Response:**
```json
{
  "articles": [
    {
      "url": "https://thehackernews.com/...",
      "title": "New ransomware targets...",
      "title_vi": "Ransomware mới tấn công...",
      "description": "...",
      "source": "The Hacker News",
      "icon": "🔐",
      "published": "2026-03-24T06:00:00Z",
      "category": "cybersecurity",
      "tags": ["Ransomware", "Zero-Day"],
      "audio_cached": true,
      "audio_url": "/api/news/audio/abc123.mp3"
    }
  ],
  "total": 15,
  "category": "cybersecurity",
  "ai_status": "idle"
}
```

---

### `POST /api/news/summarize`
Scrape an article, translate/summarize it with AI, generate MP3 via Edge-TTS, and cache the result.

**Rate limit:** 5/minute

**Request body:**
```json
{
  "url": "https://thehackernews.com/2026/03/article.html",
  "title": "New ransomware targets...",
  "lang": "en"
}
```

**Response (success):**
```json
{
  "audio_url": "/api/news/audio/abc123.mp3",
  "summary_vi": "Bài báo đề cập đến...",
  "cached": false
}
```

**Response (error, retryable):**
```json
{
  "error": "Failed to scrape article content",
  "retryable": true
}
```

**Notes:**
- If the article is already cached, returns immediately from `data/summaries/<hash>.json`
- Backend returns HTTP 500 if scraping fails or AI fails — frontend reads the error text
- `retryable: true` means the user can try again (transient error); `false` means permanent failure

---

### `POST /api/news/reprocess`
Force re-scrape and re-summarize an article, bypassing all cache (useful if previous summary was bad quality).

**Request body:**
```json
{ "url": "https://..." }
```

---

### `GET /api/news/history`
Return all articles that have been summarized in the last 7 days.

**Query params:** `category` (optional filter)

**Response:**
```json
{
  "articles": [
    {
      "url": "https://...",
      "title_vi": "...",
      "summary_vi": "...",
      "audio_url": "/api/news/audio/abc123.mp3",
      "source": "The Hacker News",
      "category": "cybersecurity",
      "created_at": "2026-03-23T10:00:00Z"
    }
  ]
}
```

---

### `GET /api/news/audio/{filename}`
Stream an MP3 file from `data/summaries/audio/`.

**Response:** `audio/mpeg` stream (supports range requests for seeking)

---

### `GET /api/news/search`
Search articles using DuckDuckGo Search API.

**Query params:** `q` (search query), `limit` (default: 20)

---

### `POST /api/news/clear-cache`
Delete all cached summary JSON and audio MP3 files.

---

### `GET /api/news/ai-status`
Return the current AI processing status for the news pipeline.

**Response:**
```json
{ "status": "idle" }
```
Status values: `idle`, `translating`, `summarizing`, `generating_audio`

## 5. Frontend-only Next.js API Routes

These files exist in `frontend-next/src/app/api/` and proxy or transform backend responses:

| Route file | Endpoint | Purpose |
|-----------|---------|---------|
| `api/chat/route.js` | `POST /api/chat` | Proxy with session-id header injection |
| `api/news/route.js` | `GET /api/news` | Proxy news fetch |
| `api/news/history/route.js` | `GET /api/news/history` | Proxy history fetch |
| `api/news/reprocess/route.js` | `POST /api/news/reprocess` | Proxy reprocess |
| `api/news/search/route.js` | `GET /api/news/search` | Proxy search |

All other `/api/*` routes are handled by the `next.config.js` wildcard rewrite directly to the backend.
