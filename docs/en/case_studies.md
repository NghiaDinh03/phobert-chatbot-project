# Enterprise Case Studies: ISO 27001 & TCVN 11930 in Vietnam

## 1. Overview

Vietnam's information security compliance landscape has evolved rapidly since 2017. Two standards dominate:

- **ISO/IEC 27001:2013/2022** — International ISMS standard, adopted voluntarily by private-sector enterprises (especially banking, fintech, telecom).
- **TCVN 11930:2017** — Vietnamese national standard defining 5 security levels for information systems, mandatory for government agencies under Decree 85/2016/NĐ-CP and Circular 03/2017/TT-BTTTT.

Key regulatory drivers:

| Regulation | Year | Scope |
|---|---|---|
| Cybersecurity Law (24/2018/QH14) | 2019 | All organizations operating in cyberspace in Vietnam |
| Decree 85/2016/NĐ-CP | 2016 | Information system security levels for state agencies |
| Circular 03/2017/TT-BTTTT | 2017 | Implementation guidance for Decree 85 |
| Circular 12/2022/TT-BTTTT | 2022 | Updated requirements for government IT system security |
| Decree 13/2023/NĐ-CP | 2023 | Personal data protection (Vietnam's "GDPR") |
| TCVN 11930:2017 | 2017 | Technical requirements for information system security by level |

As of 2025, Vietnam has approximately **200+ organizations** with active ISO 27001 certifications, concentrated in banking, telecommunications, and technology sectors. TCVN 11930 compliance is tracked by the Ministry of Information and Communications (BTTTT) across all government IT systems.

> **Note:** Exact certification counts fluctuate annually. The ISO Survey of Certifications (published by ISO) provides aggregate country data but not company-level detail for all organizations.

|---|---|

**Compliance statistics (publicly reported):**

| Percentage of Level 3+ systems with completed security assessments | ~65% | 2023 |
| Ministries/agencies with dedicated cybersecurity units | 22/30 ministries | 2023 |
| Provincial-level IT systems assessed against TCVN 11930 | All 63 provinces | 2022–2023 |

> **Note:** These statistics are based on BTTTT annual reports and National Cybersecurity Center (NCSC) publications. Exact figures vary between reporting periods. Updated 2024 data may differ.

## 6. Lessons Learned

### 6.1 Common Challenges Across Sectors

| Challenge | Banking | Healthcare | Government | Technology |
|---|---|---|---|---|
| Budget constraints | Low | **High** | **High** | Medium |
| Skilled personnel shortage | Medium | **High** | **High** | Low |
| Legacy system integration | **High** | **High** | **High** | Medium |
| Management commitment | High | Medium | Variable | High |
| Regulatory complexity (dual standards) | **High** | Medium | **High** | Medium |
| Vendor/supply chain security | Medium | Medium | **High** | **High** |

### 6.2 Success Factors

1. **Executive sponsorship** — Every successful implementation had C-level or board-level commitment. Banks with CISOs reporting directly to the CEO achieved certification 30–40% faster.
2. **Phased approach** — Organizations that started with a limited scope (e.g., data center only) and expanded achieved faster initial certification. Typical expansion: DC → core systems → all IT operations (18–36 months total).
3. **External expertise** — All major certifications involved international consulting firms (Big 4 or specialized firms like Control Risks, Protiviti) for gap analysis and implementation support.
4. **Regulatory alignment** — Organizations that mapped ISO 27001 controls to Vietnamese regulations (TCVN 11930, Circular 09/2020 for banks) from the start avoided duplicate effort.
5. **Automation investment** — GRC platforms (ServiceNow GRC, Archer, OneTrust) significantly reduced ongoing compliance burden for organizations with 50+ controls.

### 6.3 Timeline Patterns

| Phase | Typical Duration | Notes |
|---|---|---|
| Gap analysis | 2–4 months | Longer for organizations without prior ISMS |
| Risk assessment & treatment plan | 2–3 months | Requires asset inventory completion first |
| Policy & procedure development | 3–6 months | Depends on existing documentation maturity |
| Technical control implementation | 6–12 months | Largest variable; depends on infrastructure changes needed |
| Internal audit | 1–2 months | Often conducted by external internal auditors |
| Certification audit (Stage 1 + Stage 2) | 2–3 months | Stage 1: documentation review; Stage 2: implementation evidence |
| **Total (gap analysis to certification)** | **12–24 months** | Banks: 12–18 months; Government: 18–24 months; SMEs: 8–12 months |

### 8.2 Lessons from Regional Peers

**Singapore — MAS Technology Risk Management (TRM) Guidelines:**
Singapore's Monetary Authority (MAS) TRM guidelines are the gold standard in ASEAN for financial sector cybersecurity regulation. Vietnamese banks implementing ISO 27001 increasingly reference MAS TRM as a benchmark, particularly for cloud security and API risk management.

**Malaysia — Government ISMS Mandate (MyMIS):**
Malaysia's government ISMS mandate (Malaysian Public Sector Information Security Management System) is a useful comparison for Vietnam's TCVN 11930 adoption. Both countries mandate security standards for government IT systems, but Malaysia has been implementing since 2010, giving Vietnam a useful reference for scaling government compliance.

**Thailand — PDPA 2019 Impact:**
Thailand's Personal Data Protection Act (2019) drove a surge in ISO 27001 certifications across the Thai banking and telecom sectors. Vietnam is experiencing a similar effect following Decree 13/2023/NĐ-CP on personal data protection.
